## Обзор

Beacon — самостоятельно размещаемый (self-hosted) зашифрованный мессенджер. Все данные пользователя защищены сквозным шифрованием; сервер не имеет доступа к открытому тексту сообщений, ключам или медиапотокам звонков. Приложение разработано для применения в условиях противодействия: включает обнаружение активных инструментов перехвата, настраиваемое автоматическое уничтожение данных и режим-приманку, позволяющий предъявить чистое устройство при принуждении.

**Версия:** 1.038
**Минимальная версия Android:** 8.0 (API 26)
**Target SDK:** 34
**Языки:** Kotlin (Jetpack Compose), Python 3 (сервер)

## Функциональность

### Сообщения
- Зашифрованная переписка один-на-один
- Голосовые сообщения
- Передача изображений и файлов (чанкованная, до 6 МБ на пакет)
- Исчезающие сообщения с настраиваемым TTL
- Реакции, редактирование, удаление сообщений
- Ответы с цитированием
- Статусы прочтения и доставки
- Черновики сообщений

### Группы
- Зашифрованные групповые чаты
- Отдельный ключ AES-256 на группу, зашифрованный для каждого участника
- Роли администратора и участника
- Описание группы и эмодзи-аватар
- Ротация ключа при исключении участника

### Каналы
- Каналы только для публикации (администратор → подписчики)
- Подписка через deep link (`beacon://channel?...`)
- Кэш до 200 постов на канал

### Голосовые и видеозвонки
- Peer-to-peer аудио и видео через WebRTC
- TURN-ретрансляция по TCP (порт 4433) для обхода NAT
- Сигнализация через сервер, медиапоток не проходит через сервер

### Контакты и приглашения
- Инвайт-коды с ECDSA-подписью и TTL 7 дней
- Обмен через QR-код
- Экран верификации отпечатка ключа
- Аватар контакта (128×128 JPEG)

### Безопасность
- Сквозное шифрование (ECDH + AES-256-GCM)
- Прямая секретность через одноразовые предварительные ключи (prekey bundles)
- Двойное шифрование данных на устройстве (Storage Master Key)
- Биометрическая разблокировка
- Пароль-паника (запускает уничтожение данных при входе)
- Три уровня уничтожения данных: SOFT / HARD / NUCLEAR
- Режим-приманка (фиктивный аккаунт после уничтожения данных)
- Мёртвая рука (Dead Man's Switch) с настраиваемым интервалом
- Параноидный режим (подавление логов, HTTP-оповещение об угрозе)
- Обнаружение вторжения (прокси, пользовательский CA, VPN, ADB, режим разработчика)
- Антиотладка
- Защита от скриншотов (`FLAG_SECURE`)
- Обнаружение root
- Привязка сертификата (certificate pinning)

### Резервное копирование
- Полный зашифрованный бэкап и восстановление
- AES-256-GCM + PBKDF2-SHA256 (100 000 итераций)
- Экспорт через системный диалог выбора файлов

## Системные требования

### Клиент
- Android 8.0+ (API 26)
- Google Play Services (опционально, для push-уведомлений)
- Разрешения на микрофон и камеру для звонков

### Сервер
- Python 3.10+
- Библиотеки: `websockets`, `cryptography`
- Доступный IP-адрес с открытым портом 443 (WSS) или другим настроенным
- TURN-сервер для обхода NAT (рекомендуется coturn)

## Сборка

### Требования
- Android Studio Hedgehog (2023.1) или новее
- JDK 17
- Android SDK с API 34

### Шаги

```bash
# Перейти в директорию проекта
cd TEST2

# Собрать отладочный APK
./gradlew assembleDebug

# Собрать релизный APK (требует настройки подписи в app/build.gradle.kts)
./gradlew assembleRelease
```

Релизный APK сохраняется в `app/build/outputs/apk/release/`.

### Настройка подписи

Укажите свой keystore в `app/build.gradle.kts`:

```kotlin
signingConfigs {
    create("release") {
        storeFile = file("your-keystore.jks")
        storePassword = System.getenv("KEYSTORE_PASSWORD")
        keyAlias = "messenger"
        keyPassword = System.getenv("KEY_PASSWORD")
    }
}
```

> **Важно:** Никогда не храните пароли keystore в коде. Используйте переменные окружения или `local.properties`.

## Развёртывание сервера

```bash
cd ForEXP

# Установить зависимости
pip install websockets cryptography

# Настроить переменные окружения
export CHANNEL_ADMIN_SECRET="ваш-секрет-здесь"
export PORT=8765                          # опционально, по умолчанию 8765

# Запустить
python server.py
```

### С TLS (рекомендуется для продакшна)

Разместите сервер за обратным прокси (nginx, Caddy), который завершает TLS и пробрасывает WebSocket-соединения. Либо передайте SSL-контекст напрямую в `server.py`.

### TURN-сервер (coturn)

Необходим для WebRTC-звонков через NAT:

```
listening-ip=<IP_СЕРВЕРА>
relay-ip=<IP_СЕРВЕРА>
listening-port=4433
lt-cred-mech
user=<ИМЯ_ПОЛЬЗОВАТЕЛЯ>:<ПАРОЛЬ>
realm=beacon
```

## Быстрый старт

1. Развернуть сервер и указать его адрес в `NetworkConfig.kt`.
2. Собрать и установить APK.
3. Открыть приложение, зарегистрировать имя пользователя и пароль.
4. Поделиться инвайт-кодом с контактом (`beacon://invite?...`).
5. После того как контакт добавит вас, начать переписку.

## Безопасность

Beacon использует многоуровневую модель защиты:

| Уровень | Механизм |
|---|---|
| Транспорт | WSS (TLS 1.2+) с привязкой сертификата |
| Сквозное шифрование | ECDH (P-256) + AES-256-GCM |
| Прямая секретность | Одноразовые prekey-пакеты (X3DH-схема) |
| Данные на устройстве (L1) | EncryptedSharedPreferences + AndroidKeyStore |
| Данные на устройстве (L2) | Storage Master Key, PBKDF2-SHA256 300 000 итераций |
| Аутентификация | PBKDF2-SHA256 100 000 итераций |
| Подписи ключей | ECDSA (SHA256withECDSA) |
