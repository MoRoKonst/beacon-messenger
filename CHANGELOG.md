# Changelog

## [Unreleased]

### Исправления

- **Анонимная маршрутизация теперь охватывает весь трафик** — typing, read, delivered,
  session_reset, реакции, редактирование, удаление сообщений, таймер исчезающих сообщений,
  чанки изображений/файлов/видео, групповые сообщения (group_create, group_message,
  group_reaction, group_member_removed, group_key_rotation, group_invite_accepted),
  WebRTC call_end — всё идёт через `anon_message → token` если токены есть.
  Прямая отправка по fingerprint остаётся только как fallback до первого обмена токенами.

- **session_init теперь тоже анонимен** — если токены уже получены через mailbox,
  X3DH handshake отправляется через `anon_message`, а не напрямую по fingerprint.
  Сервер не видит пару A → B ни разу после первого mailbox-обмена.

- **Задержка после первого контакта устранена** — после получения токенов от контакта
  `contactMailboxTag` очищается, и последующие сообщения идут через быстрый anon-маршрут
  вместо ожидания 30-секундного polling'а mailbox. Дополнительно: если токены уже есть,
  `send()` минует mailbox-путь даже если тег ещё не очищен.

- **Mailbox polling каждые 30 секунд** — ранее `pollMailbox` вызывался только при
  подключении; теперь периодически повторяется пока клиент онлайн, чтобы не пропускать
  новые первые контакты.

- **Устранён бесконечный цикл BAD_DECRYPT / session_reset** — оба клиента одновременно
  вызывали `requestPrekeyBundle` после `session_reset`, инициируя X3DH с обеих сторон
  и создавая несовпадающие сессии. Исправлено: `session_reset` больше не запрашивает
  bundle — инициатива остаётся за тем, кто отправил первое сообщение.

- **Контакт сохраняется при обмене только токенами** — ранее `addContact` не вызывался
  если первым приходил системный пакет `__beacon_tokens__`.

- **Без Orbot приложение подключается немедленно** — таймаут ожидания Orbot уменьшен
  с 90 до 12 секунд; добавлен обработчик `onTorError` чтобы `connect()` вызывался
  сразу при ошибке, не зависая в ожидании.

- **Токены отправляются только через анонимную почту** — убран прямой fingerprint-fallback
  в `sendAnonTokensTo`; если нет анон-токена для отправки — ждём mailbox-обмена.

## Предыдущие версии

### Анонимная маршрутизация (токены + mailbox + cover traffic)

- Токенная маршрутизация: клиент регистрирует пул из 50 одноразовых токенов;
  сообщения адресуются по токену, а не по fingerprint.
- Анонимный mailbox: первое сообщение кладётся в mailbox по тегу из инвайт-кода;
  получатель запрашивает всегда ровно 20 тегов (реальные + фейки).
- Constant-rate cover traffic (режимы: выкл / умеренный 5 сек / агрессивный 1 сек).
- Поддержка Tor hidden service через Orbot SOCKS.

### WebRTC видеозвонки

- Библиотека `stream-webrtc-android:1.3.10`, Camera1Enumerator, VP8+H264 Baseline.
- Фикс краша: `onTrack` и `handleCallEnd` диспатчатся на main thread.
- TURN через TCP порт 4433 (UDP заблокирован провайдером).

### Двойное шифрование at-rest (SMK)

- Storage Master Key (256-bit AES), хранится через PBKDF2 и AndroidKeyStore.
- Защищает EC private key, групповые ключи, блобы сообщений.
- Прозрачная миграция legacy данных.
